**2\. แผนการทดสอบและ CI/CD (Testing & CI/CD Plan)**

 **2.1 Test Plan Document**

* รายละเอียด: แผนการทดสอบของระบบ (Unit, Integration, Scenario)

   ผู้รับผิดชอบ: **QA**

 **2.2 Unit Tests**

* รายละเอียด:  
  * ใช้ Jest  
  * ≥ 15 Test Cases  
  * Coverage ≥ 80%

   ผู้รับผิดชอบ: **DEV**

 **2.3 Integration Tests**

* รายละเอียด:  
  * ทดสอบการทำงานร่วมกันของระบบ  
  * ≥ 5 Test Suites

ผู้รับผิดชอบ: **DEV**

**2.4 Test Cases Spreadsheet**

* รายละเอียด:  
  * Test Case ID  
  * Description  
  * Input  
  * Expected Output  
  * Status

  ผู้รับผิดชอบ: **QA**

 

 **2.5 Test Coverage Report**

* รายละเอียด:  
  * รายงาน Coverage จาก Jest  
  * (Screenshot / HTML Report)  
  * ≥ 80%

  ผู้รับผิดชอบ: **DEV**

 **2.6 UAT Scenarios**

* รายละเอียด:  
  * 3–5 สถานการณ์ทดสอบจริง  
  * ทดสอบร่วมกับ PO

  ผู้รับผิดชอบ: **QA**

 

 

 

**3\. Definition of Done**

งานจะถือว่า “เสร็จสมบูรณ์” เมื่อ:

* การทดสอบทั้งหมดผ่าน  
* ความครอบคลุมของโค้ด (Coverage) ≥ 80%  
* Integration Test ครบตามที่กำหนด  
* ไม่มีข้อผิดพลาดร้ายแรง (Critical Bug)  
* มีเอกสารครบ ได้แก่ Test Plan, Test Cases, Coverage Report และ UAT


 

**4\. บทบาทใน Sprint**

| บทบาท | หน้าที่ |
| :---- | :---- |
