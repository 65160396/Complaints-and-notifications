### 6.2 ความเสี่ยงใหม่ที่ระบุ

- ความเสี่ยงจาก การ deploy อัตโนมัติ (CI/CD)  
→ หาก push ผิด branch อาจ deploy code ที่ยังไม่พร้อมขึ้น production  

- ความเสี่ยงด้าน Security ของระบบ (Authentication / Data access)  
→ ต้องมีการตรวจสอบสิทธิ์ผู้ใช้งานให้เหมาะสมกับ role  

- ความเสี่ยงจาก Real-time notification ยังไม่ใช้ WebSocket เต็มรูปแบบ  
→ อาจเกิด delay ในการแจ้งเตือน  

- ความเสี่ยงด้าน Scalability  
→ หากมีผู้ใช้งานจำนวนมาก ระบบ (Render / Railway) อาจรองรับไม่เพียงพอ  

- ความเสี่ยงจาก Frontend (Next.js SSR + localStorage)  
→ เคยเกิด error ต้องมีการเช็ค typeof window ทุกจุดที่ใช้
