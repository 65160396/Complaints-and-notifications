# UAT (User Acceptance Testing) Scenarios

## Scenario 1: นิสิตแจ้งปัญหา ติดตามสถานะ และประเมินผล
**Objective:** นิสิตสามารถแจ้งปัญหา ติดตามสถานะ และประเมินความพึงพอใจได้สำเร็จ

**Steps:**
1.	เปิดเบราว์เซอร์ไปที่ localhost:3000
2.	กรอกอีเมล @buu.ac.th และรหัสผ่าน กดปุ่ม 'เข้าสู่ระบบ'
3.	กดปุ่ม 'แจ้งเรื่องใหม่' เลือกหมวดหมู่ 'อาคารสถานที่'
4.	เลือกสถานที่จาก Dropdown ตามลำดับชั้น คณะ → อาคาร → ชั้น
5.	กรอกรายละเอียดปัญหา แนบรูปภาพ ตรวจสอบ Preview แล้วกด 'ส่งคำร้อง'
6.	บันทึกเลขที่คำร้อง (Ticket ID) ที่ระบบออกให้
7.	ไปที่เมนู 'คำร้องของฉัน' ตรวจสอบ Timeline สถานะ 'รอดำเนินการ'
8.	กดปุ่ม 'ยกเลิกคำร้อง' กรอกเหตุผล ยืนยันการยกเลิก
9.	เปิดคำร้องสถานะ 'เสร็จสิ้น' กดปุ่ม 'ประเมินความพึงพอใจ'
10.	ให้คะแนน 4 ดาว กรอกข้อเสนอแนะ กดส่ง ตรวจสอบว่าแก้ไขซ้ำไม่ได้

**Expected Outcome**
- ✓  เข้าสู่ระบบสำเร็จ ระบบ Redirect ไปหน้า Dashboard ของนิสิตโดยอัตโนมัติ
- ✓  Dropdown สถานที่โหลดภายใน 2 วินาที แสดงลำดับชั้นถูกต้อง
- ✓  สามารถแนบรูปภาพและเห็น Preview ก่อนส่ง
- ✓  ระบบออก Ticket ID และแสดงข้อความ 'ส่งเรื่องสำเร็จ' ทันที
- ✓  หลังยืนยันยกเลิก สถานะเปลี่ยนเป็น 'ยกเลิกแล้ว' ทันที
- ✓  ปุ่มประเมินปรากฏเฉพาะคำร้องสถานะ 'เสร็จสิ้น' เท่านั้น
- ✓  หลังส่งประเมินแล้ว ไม่สามารถแก้ไขซ้ำได้

**ผู้ทดสอบ:** Product Owner

## Scenario 2: บุคลากรแจ้งเรื่องร้องเรียนเชิงนโยบายพร้อมกำหนดความเร่งด่วน
**Objective:** บุคลากรสามารถส่งเรื่องร้องเรียนเชิงนโยบายและกำหนดระดับความเร่งด่วนได้ถูกต้อง 

**Steps:**
1.  เข้าสู่ระบบด้วยบัญชีบุคลากร ตรวจสอบว่า Dashboard แสดงภาพรวมหน่วยงาน
2.	กดปุ่ม 'แจ้งเรื่องใหม่' เลือกหมวดหมู่ 'เชิงนโยบาย / การบริหาร'
3.	ระบุสถานที่หรือหน่วยงานที่เกี่ยวข้องจาก Dropdown
4.	กรอกรายละเอียดปัญหาอย่างละเอียด แนบไฟล์หลักฐาน
5.	เลือกระดับความเร่งด่วนเป็น 'ด่วน' กดส่งคำร้อง
6.	ตรวจสอบ Ticket ID และป้ายสีระดับความเร่งด่วนที่แสดง
7.	ไปที่ตั้งค่าการแจ้งเตือน ปิดแจ้งเตือนในระบบ เปิดทาง Email บันทึก

**Expected Outcome**
- ✓  Dashboard ของบุคลากรแสดงภาพรวมหน่วยงาน (ต่างจาก Dashboard นิสิต)
- ✓  ระดับความเร่งด่วน 'ด่วน' แสดงป้ายสีที่ชัดเจน (สีเหลือง)
- ✓  Ticket ID ออกทันทีพร้อมข้อความยืนยัน
- ✓  สามารถตั้งค่าช่องทางการแจ้งเตือนได้อิสระตามต้องการ
- ✓  ระบบบันทึกและจำการตั้งค่าการแจ้งเตือนไว้ถูกต้อง

##  Scenario 3: สโมสรคณะคัดกรอง แก้ไขหมวดหมู่ และส่งต่องาน
**Objective:** สโมสรคณะสามารถคัดกรองคำร้อง แก้ไขหมวดหมู่ และส่งต่อให้ส่วนกลางได้ถูกต้อง

**Steps:**
1.	เข้าสู่ระบบ ตรวจสอบว่า Dashboard แสดงเฉพาะคำร้องของคณะตนเองเท่านั้น
2.	เปิดคำร้องที่มีหมวดหมู่ผิด อ่านรายละเอียดและดูรูปภาพ
3.	กดปุ่ม 'แก้ไขหมวดหมู่' เปลี่ยนเป็นหมวดหมู่ที่ถูกต้อง กด 'ยืนยัน'
4.	ปรับระดับความเร่งด่วนตามความเหมาะสม กด 'บันทึก'
5.	กดปุ่ม 'รับเรื่อง' ตรวจสอบว่าสถานะเปลี่ยนและชื่อสโมสรถูกบันทึก
6.	เปิดคำร้องที่เกินขอบเขตคณะ กดปุ่ม 'ส่งต่อกองกลาง' กรอกเหตุผล ยืนยัน
7.	ตรวจสอบว่าตัวเองยังดู (View Only) ได้ แต่ปุ่มแก้ไขถูกปิด
8.	พยายามเข้าดูคำร้องของคณะอื่นผ่าน URL โดยตรง

**Expected Outcome**
- ✓  Dashboard แสดงเฉพาะคำร้องของคณะตนเอง ไม่มีข้อมูลคณะอื่น
- ✓  สามารถแก้ไขหมวดหมู่ได้ ระบบบันทึกประวัติการแก้ไข
- ✓  หลังกด 'รับเรื่อง' สถานะเปลี่ยนทันที ชื่อสโมสรปรากฏในรายละเอียด
- ✓  หลังส่งต่อแล้ว ปุ่มแก้ไขถูกปิดอัตโนมัติ เหลือเพียงโหมด View
- ✓  การเข้าถึงคำร้องคณะอื่นถูกปฏิเสธพร้อมข้อความแจ้งเตือน

## Scenario 4: เจ้าหน้าที่มหาวิทยาลัยรับงาน มอบหมายทีมช่าง และจบงาน
**Objective:** เจ้าหน้าที่สามารถรับงาน มอบหมายทีมช่าง และจบงานพร้อมยืนยันอีเมลอัตโนมัติ

**Steps:**
1.	เข้าสู่ระบบ ตรวจสอบ Dashboard ระดับมหาวิทยาลัยที่แสดงคำร้องจากทุกคณะ
2.	เปิดคำร้องที่ถูกส่งต่อมา อ่านสรุปเหตุผลที่สโมสรกรอกไว้
3.	กดปุ่ม 'รับเรื่อง' ตรวจสอบสถานะและชื่อที่บันทึก
4.	กดปุ่ม 'มอบหมายงาน' เลือกทีมช่างหรือหน่วยงานที่รับผิดชอบ
5.	ตรวจสอบว่าผู้รับผิดชอบได้รับการแจ้งเตือน
6.	กดปุ่ม 'จบงาน' แนบรูปภาพหลังดำเนินการ (After Photo) กรอกสรุปผล ยืนยัน
7.	จับเวลาตรวจสอบว่านิสิตผู้แจ้งได้รับอีเมลภายใน 60 วินาที
8.	ตรวจสอบว่าปุ่ม 'ประเมินความพึงพอใจ' ปรากฏในบัญชีนิสิต

**Expected Outcome**
- ✓  Dashboard แสดงคำร้องจากทุกคณะที่ส่งต่อมา
- ✓  หลังรับเรื่อง ชื่อเจ้าหน้าที่บันทึกในระบบ และเจ้าหน้าที่คนอื่นรับซ้ำไม่ได้
- ✓  ผู้รับผิดชอบที่มอบหมายได้รับการแจ้งเตือนจากระบบ
- ✓  สถานะเปลี่ยนเป็น 'เสร็จสิ้น' ทันทีหลังจบงาน
- ✓  อีเมลส่งถึงนิสิตผู้แจ้งภายใน 60 วินาที
- ✓  แบบประเมินปรากฏในบัญชีนิสิตเท่านั้น

## Scenario 5: ผู้ดูแลระบบจัดการข้อมูลหลัก สิทธิ์ผู้ใช้ และออกรายงาน
**Objective:** ผู้ดูแลระบบสามารถบริหารข้อมูลหลัก จัดการสิทธิ์ผู้ใช้ และส่งออกรายงานได้ครบถ้วน

**Steps:**
1.	เข้าสู่ระบบ Admin Portal ตรวจสอบ Dashboard ภาพรวมทั้งระบบ
2.	ไปที่ 'จัดการข้อมูลหลัก' เพิ่มอาคารใหม่ชื่อ 'อาคารทดสอบ UAT' บันทึก
3.	สลับไปแท็บนิสิต Refresh หน้า ตรวจสอบว่า Dropdown สถานที่มีอาคารใหม่
4.	กลับมา Admin ไปที่ 'จัดการผู้ใช้' ค้นหาบัญชีนิสิต Test
5.	เปลี่ยน Role จาก 'นิสิต' เป็น 'สโมสรคณะ' บันทึก
6.	ตรวจสอบ Audit Log ว่าบันทึก ใคร / เมื่อไหร่ / เปลี่ยนอะไร ครบถ้วน
7.	ไปที่ 'จัดการการแจ้งเตือน' สร้างประกาศฉุกเฉิน กำหนดกลุ่มเป้าหมายทุกคน ส่ง
8.	ตรวจสอบว่าบัญชีนิสิต Test ได้รับแจ้งเตือนบน Dashboard
9.	ไปที่ 'รายงาน' เลือกช่วงเวลา 1 เดือน Export เป็น CSV และ PDF
10.	ดาวน์โหลดทั้ง 2 ไฟล์ ตรวจสอบว่าเปิดได้และข้อมูลถูกต้อง
11.	กด 'ออกจากระบบ' กด Back ตรวจสอบว่าไม่สามารถกลับเข้าระบบได้

**Expected Outcome**
- ✓  อาคารใหม่ปรากฏใน Dropdown ของนิสิตทันที (หรือหลัง Refresh ครั้งเดียว)
- ✓  Role เปลี่ยนสำเร็จ ผู้ใช้เห็นเมนูตาม Role ใหม่ทันที
- ✓  Audit Log บันทึกครบ: ใคร / เมื่อไหร่ / เปลี่ยนอะไร
- ✓  ประกาศส่งถึงผู้ใช้ทุกคนและปรากฏบน Dashboard ภายใน 1 นาที
- ✓  Export CSV และ PDF สำเร็จ ข้อมูลถูกต้องและเปิดได้
- ✓  หลัง Logout กด Back ไม่สามารถกลับเข้าระบบได้

**วันที่ทดสอบ:** 

**ผู้ทดสอบ:** QA Lead
