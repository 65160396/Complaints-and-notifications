 **5\. Definition of Done (DoD)**

งานจะถือว่าเสร็จเมื่อ:

* ฟังก์ชันทำงานได้ตาม User Story

* เชื่อมฐานข้อมูลได้

* ทดสอบแล้วไม่มี error สำคัญ

* โค้ดถูกบันทึกใน repository

* PO ตรวจสอบแล้ว

### **7\. เหตุผลในการวางแผน Sprint**

**Sprint 1 มุ่งเน้นสร้าง Minimum Viable Product (MVP)**

เพื่อให้ระบบสามารถทำงานได้ครบตาม Core Flow หลักของผู้ร้องเรียน ได้แก่

เข้าสู่ระบบ → ส่งคำร้อง → ระบุสถานที่ → ติดตามสถานะ → ออกจากระบบ

หลังจากนั้นจึงพัฒนาเพิ่มเติมใน Sprint ถัดไป ได้แก่

* ระบบแจ้งเตือน

* การกำหนดระดับความเร่งด่วน

* รายงานสรุปข้อมูล

* ฟีเจอร์เสริมอื่น ๆ
